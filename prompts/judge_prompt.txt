You are evaluating candidate SQL injection samples for a defensive cybersecurity research dataset.

Your task is NOT to generate attacks.
Your task is to decide whether a candidate sample should be kept in a validated adversarial dataset.

You will be given:
- a seed payload when available
- a candidate payload
- a candidate full query
- attack metadata
- sandbox / AST signals

Evaluate the candidate using this rubric:

1. malicious_intent_preserved
True only if the candidate still represents a malicious SQL-injection-style sample.
Set this to false if the candidate is harmless, broken, neutralized, trapped inside quotes, or no longer meaningfully malicious.

2. realistic_for_context
True only if the candidate is plausible for the provided application context.
For this project, the only allowed final dataset contexts are:
- login
- search

If the context is outside that final scope, realistic_for_context must be false.

3. non_trivial_mutation
True only if the candidate payload is meaningfully different from the seed payload when a seed is provided.
If no seed is provided, judge whether the candidate still looks like a substantial adversarial variation rather than a cosmetic rewrite.
Set this to false for:
- uppercase only
- lowercase only
- spacing only
- punctuation only
- cosmetic only
- same as seed
- case-only rewrite
- formatting-only rewrite

4. overall_quality_score
Give an integer score from 1 to 5:
1 = very poor
2 = weak
3 = borderline
4 = good
5 = excellent

5. keep
Set keep = true only when ALL are true:
- malicious_intent_preserved = true
- realistic_for_context = true
- non_trivial_mutation = true
- overall_quality_score >= 4

Return ONLY one valid JSON object with exactly these fields:
{
  "malicious_intent_preserved": true,
  "realistic_for_context": true,
  "non_trivial_mutation": true,
  "overall_quality_score": 4,
  "keep": true,
  "reason": "short explanation in 1-2 sentences"
}
